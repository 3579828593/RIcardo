# 期末冲刺刷题系统 — 长期产品路线图

> 本文档是 2026-06-29 头脑风暴的产出，指导未来 2-3 个月（暑假集中开发）的产品和工作流方向。
> 不代替具体功能的 SPEC，而是为每个阶段定义目标、交付物和验收标准。

## 1. 产品定位

**一句话**：面向大学生的垂直刷题平台——不只是刷题工具，而是"刷题计划 + 学习分析 + 游戏化体验"的组合。

**目标用户**：
- 短期（暑假）：班级同学，30-50 人期末复习
- 中期（秋季）：校内推广，几百人规模
- 长期：多校园、多学科

**差异化策略**：
- 主线：基础体验做到极致（快、稳、数据驱动）
- 支线：游戏化体验（像素风 RPG/肉鸽式刷题，慢慢构思）
- 内容：UGC 题目导入（解决题目来源，用户自给自足）

## 2. 技术路线

```
当前：网站 (PWA 已有基础) → 验证核心体验
  ↓
中期：PWA + 微信小程序（触达大学生群体）
  ↓
远期：验证成功后用 Flutter/React Native 出原生 App
```

**原则**：网站做快速验证，不过早投入原生 App 开发。PWA 已支持"添加到桌面"，体验接近原生。

## 3. 四阶段路线图（暑假 8 周）

### 阶段 1：快而稳（第 1-2 周）

**目标**：打开即用，3 秒内可答题，微信内不白屏。

**交付物**：同学发链接就能流畅刷题，不需要等待。

| 任务 | 说明 | 优先级 |
|------|------|--------|
| UptimeRobot 保活 | 每 5 分钟 ping 首页，解决 PythonAnywhere 冷启动 5-10 秒 | 高 |
| Service Worker 离线答题 | 离线时仍可答题，答案缓存后联网同步 | 高 |
| 骨架屏 + 渐进加载优化 | 首屏骨架屏，数据加载后平滑过渡 | 中 |
| 加载性能分析 | Lighthouse 评分 ≥ 80，FCP < 2 秒 | 中 |
| Playwright E2E 测试框架 | 搭建端到端测试，覆盖核心用户路径 | 中 |

**验收标准**：
- [ ] 微信内打开 3 秒内显示题目
- [ ] 离线状态下可答题（答案暂存，联网同步）
- [ ] Lighthouse 性能评分 ≥ 80
- [ ] Playwright E2E 覆盖：答题→提交→下一题→进度更新

**工作流附带**：Playwright E2E 测试框架搭建，首次引入浏览器自动化测试。

---

### 阶段 2：自己能加题（第 3-4 周）

**目标**：不用改代码就能加题。

**交付物**：Excel 模板 → 上传 → 题库更新，管理后台基础版。

| 任务 | 说明 | 优先级 |
|------|------|--------|
| CSV/Excel 导入 | 提供 Excel 模板，支持批量导入题目 | 高 |
| Markdown 导入 | 支持用 Markdown 语法编写题目 | 中 |
| Admin Token 认证 | 管理后台需要 token 才能访问 | 高 |
| 题目管理界面 | 查看全部题目、编辑、删除、按章节筛选 | 高 |
| 导入校验 + 错误报告 | 导入时检查格式，返回错误行号和原因 | 高 |

**验收标准**：
- [ ] 下载 Excel 模板 → 填写 → 上传 → 题库新增成功
- [ ] 格式错误时返回具体错误行号和原因
- [ ] 管理后台需要 admin token 才能操作
- [ ] 支持单选/多选/判断/填空四种题型导入

**工作流附带**：CI/CD 自动部署（git push → 自动触发 Files API 部署）。

---

### 阶段 3：记住我是谁（第 5-6 周）

**目标**：换设备不丢进度，班级内可比较。

**交付物**：注册登录 + 跨设备同步 + 班级排行榜。

| 任务 | 说明 | 优先级 |
|------|------|--------|
| 用户注册/登录 | 邮箱 or 学号注册，JWT token 认证 | 高 |
| session_id → user_id 迁移 | 旧数据按 session_id 迁移到 user_id | 高 |
| 跨设备数据同步 | 登录后自动拉取服务端进度 | 高 |
| 班级/学习群组 | 创建班级、邀请码加入、群内排行 | 中 |
| 排行榜 | 按答题数/正确率/连续天数排序 | 中 |

**验收标准**：
- [ ] 注册 → 登录 → 换设备登录 → 进度完整恢复
- [ ] 旧 session_id 数据自动迁移到新账号
- [ ] 创建班级 → 同学用邀请码加入 → 看到班级排行榜
- [ ] 排行榜按答题数和正确率双维度排序

**工作流附带**：多项目工作流管理——SDD 模板可复用化，提取为通用脚手架。

---

### 阶段 4：告诉我该学什么（第 7-8 周）

**目标**：系统告诉你哪里薄弱、该复习什么。

**交付物**：知识点雷达图 + 错题趋势分析 + 复习计划推荐。

| 任务 | 说明 | 优先级 |
|------|------|--------|
| 知识点掌握度雷达图 | 按章节/知识点维度可视化掌握程度 | 高 |
| 错题趋势分析 | 时间轴展示错题数量变化、薄弱知识点 | 高 |
| 复习计划推荐 | 基于错题和掌握度，推荐"今天该复习什么" | 中 |
| 学习报告 | 周/月学习报告（答题数、正确率、进步趋势） | 中 |
| 数据导出 | 导出答题记录为 CSV/Excel 供个人分析 | 低 |

**验收标准**：
- [ ] 雷达图正确反映各章节掌握度
- [ ] 错题趋势图可按时间范围筛选
- [ ] 复习推荐至少覆盖前 5 个薄弱知识点
- [ ] 学习报告可分享（生成图片或链接）

**工作流附带**：工作流工具化——生成通用 SDD 脚手架脚本，可快速初始化新项目的 SDD 文件体系。

## 4. 游戏化支线（秋季课余构思）

不阻塞主线，利用课余时间慢慢构思和原型验证。

| 层级 | 内容 | 预估时间 | 何时启动 |
|------|------|----------|----------|
| L2 机制级 | 连击/血条/关卡，答对加分答错掉血 | 2-3 周 | 秋季 |
| L3 结构级 | 知识点=技能树，章节=地图，错题=Boss战 | 4-6 周 | L2 验证后 |

**构思方向**（待细化）：
- 像素风 RPG 视觉（已有 Press Start 2P 字体基础）
- 肉鸽式随机出题（每次刷题路径不同）
- 知识点技能树（解锁式学习）
- Boss 战（章节综合题挑战）

## 5. 工作流体系演进路线

工作流不单独做，而是随产品功能同步演进：

| 产品阶段 | 工作流附带 | 产出 |
|----------|------------|------|
| 阶段 1 | Playwright E2E 框架 | 浏览器自动化测试能力 |
| 阶段 2 | CI/CD 自动部署 | git push → 自动上线 |
| 阶段 3 | SDD 模板可复用化 | 多项目工作流管理 |
| 阶段 4 | 通用 SDD 脚手架 | 可快速初始化新项目 |

**远期愿景**：工作流体系本身可以成为一个开源工具——"给个人开发者用的 AI 辅助开发工作流框架"。

## 7. 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| PythonAnywhere 免费层不够用 | 中 | 服务中断 | 阶段 3 前评估升级到付费层（$5/月） |
| 暑假开发进度落后 | 高 | 阶段缩减 | 每阶段有"最小可交付"和"完整版"两档 |
| 游戏化开发复杂度超预期 | 中 | 支线延期 | 支线不阻塞主线，可无限推迟 |
| 用户增长后 SQLite 性能瓶颈 | 低 | 响应变慢 | 几百用户级别 SQLite 够用，千级再考虑迁移 |
